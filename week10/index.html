MDN: Validating forms

Validation done in browser is client-side validation, validation done by server is server-side validation.

client-side validation should not be considered an exhaustive security measure and should always be accompanied by server-side validation.

Three main reasons for client-side validation: making sure we're getting the right data in the right format, protecting users' data, and protecting ourselves from malicious users.

Two types of client-side validation: built-in form validation and javascript validation

Bulit-in form validation: required, minlength, maxlength, min, max, type, and pattern
