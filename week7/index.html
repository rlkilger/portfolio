Javascript: Novice to Ninja Ch. 11 - Further Functions

A function's length property returns how many parameters it has.

call() method can be used to set the value of this inside a function to an object that is provided as the first argument - function sayHello(){return `Hello, my name is ${ this.name }`;} clark = {name:Clark}; sayhello.call(clark) returns "Hello, my name is Clark"

Additional parameters go after first parameter that is defining the object - function sayHello(greeting='Hello'){return `${ greeting }, my name is ${ this.name }`;} sayHello.call(clark, "Hi") returns "Hi, my name is Clark"

You can add properties to a function much like you can to an object - square.description = 'Squares a number that is provided as an argument'

Can store results of a function in a cache property to save from recomputing results - function square(x){square.cache = square.cache || {}; if (!square.cache[x]) {square.cache[x] = x*x;} return square.cache[x]} square.cache; returns {"3": 9, "-11": 121}

Immediately Invoked Function Expression – or IIFE (pronounced 'iffy') - anonymous function that is invoked as soon as it’s defined - (function(){const temp = 'World'; console.log(`Hello ${temp}`);})(); immediately returns "Hello World

Creating a variable inside an IIFE makes it temporary - let a = 1; let b = 2; (()=>{const temp = a; a = b; b = temp;})(); - this can be used to switch the values of a and b, and temp will not exist after the function executes

The above example is not the most efficient way to swap values - let [a,b] = [1,2]; [a,b] = [b,a];

An IIFE can be used to set up any initialization code that there’ll be no need for again. Because the code is only run once, there’s no need to create any reusable, named functions, and all the variables will also be temporary.

This function prints a welcome message to the console - (function() {cons const date = new Date(),today = days[date.getDay()]; console.log(`Welcome back ${name}. Today is ${today}`);})(); - immediately prints 'Welcome back Peter Parker. Today is Tuesday' to the console

All of these features can also be acheived by placing the code inside a block ( {--code goes here--} )

IIFEs are useful when wanting to use strict mode - (function() {'use strict'; // All your code would go inside this function})();

Functions can redefine themselves - function party(){console.log('Wow this is amazing!'); party = function(){console.log('Been there, got the T-Shirt');}} - first will log 'Wow this is amazing!', then it will log 'Been there, got the T-Shirt'

If the function is assigned to a new variable, that variable will always display the first message and will update the original function

Any properties assigned to the function before it was redefined will be lost

Recursive function - calls itself inside the function until a certain condition is met.
