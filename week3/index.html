Javascript: Navice to Ninja Ch. 5 - Objects

Arrays and functions are objects that are built into the language

Object literal - object that is created directly in the language by wrapping all its properties and methods in curly braces, they make it possible to create an object without having to define a class

If object name is not a valid name for a variable(has space, starts with number, etc), use quotes(but it's preferable to use valid variable names)

Object can be changed even when created using const, and properties don’t always appear in the order they were entered

Create empty object - const object = {}

ES6 way of assigning properties to existing object - variable name becomes the key

Example - const name = 'Iron Man'; const realName = 'Tony Stark'; const ironMan = {name, realName};

Property value can be a Javascript expression - const bewitched = true; const captainBritain = { name: 'Captain Britain', hero: bewitched ? false : true }; returns {name: 'Captain Britain', hero: false}

Check if object has a specific property - 'city' in superman; or superman.city !== undefined; or superman.hasOwnProperty('city');returns true or false

in and !== undefined return true when property is inherited and .hasOwnProperty() does not

Loop through object properties - for(const key in superman) {console.log(key + ":if(superman.hasOwnProperty(key)){console.log(key + ": " + superman[key]);}}

Object.keys() returns an array of the object's keys

Object.values() returns an array of the object's values

Object.entries() returns an array or the object's key-value pairs

Use delete to remove property - delete superman.fly; returns true if property was deleted, false if not found

Can nest objects using curly braces

If variable is assigned to an object that already exists, changes made using either reference will affect the same object

Namespacing - use object literal to create a namespace for groups of related functions

JSON - JavaScript Object Notation

Property names must be double-quoted

Permitted values are double-quoted strings, numbers, true, false, null, arrays and objects

Functions are not permitted values

.parse() - takes a string of data in JSON format and returns a JavaScript object

.stringify() - takies JavaScript object and returns a string of JSON data(methods are ignored)

Can add a space argument that adds new lines between each key-value pair

RegExp Object (regular expression) - search strings for matches to the pattern

Search for string that contain - const pattern = /[a-zA-Z]+ing$/; or const pattern = new RegExp('[a-zA-Z]+ing');(second way can use strings

.test() returns true if pattern is found - pattern.test('joke'); returns false, pattern.test('joking'); returns true, pattern.test('jokingly'); returns false

.exec() - returns an array containing the first match found, or null if there aren’t any matches

pattern.exec('joking'); returns ['joking', index: 0, input: 'joking']

String pattern - const pattern = /string/

Charachter group - any one of the characters inside the brackets - const vowels = /[aeiou]/

Dash - any characters between - /[A-Z]/, /[0-9]/

Greedy patterns return longest matching string, while lazy paterns(+?) return shortest.

Pattern example - const pdf = /.*\.pdf$/; searches for pdf files(if they end in .pdf)

String methos examples:

'JavaScript'.match(/[aeiou]/); = ['a'] (Returns first vowel)

'JavaScript'.match(/[aeiou]/g); = ['a', 'a', 'i'] (Returns an array of all the vowels)

"I'm learning JavaScript".search(/java/i); = 13 (Returns location of first match or -1 if there is no match)

'JavaScript'.replace(/[aeiou]/ig,'*'); = 'J*v*Scr*pt' (Returns string with any matches replaced by another string)

Javascript: Novice to Ninja Ch. 6 - The Document Object Model

Assign node to a variable - const body = document.body;

.nodeName finds name of element - body.nodeName returns "BODY"

document.images - returns a node list of all the images contained in the document

document.links - returns a node list of all the "a" elements and "area" elements that have an href attribute

Document.anchors - returns a node list of all the "a" elements that have a name attribute

document.forms - returns a node list of all the forms in the document

Node lists are not arrays, but can be accessed using index notation, have a length, and you can iterate through them.

Turn node list into an array - const imageArray = Array.from(document.images); or const imageArray = [...document.images];

Can use with CSS pseudo-selectors - const wonderWoman = document.querySelector('li:last-child');

Can use querySelector() on other elements - const ul = document.querySelector('ul#roster'); (assigns ul to a variable) then const batman = ul.querySelector('li#bats'); (uses querySelector() on the variable ul)

.childNodes - returns list of all child nodes of the node that is referenced including text from white space

.children - returns list of element nodes, excludes text

Other properties - .firstChild, .lastChild, .parentNode, .nextSibling, .previousSibling

.textContent - returns the text content of an element as a string, including "script" and "style" elements(.innerText only returns “human-readable” elements

.getAttribute() - returns value of attribute - wonderWoman.getAttribute('id'); or wonderwoman.id

.setAttribute() - overwrites all previously set classes - wonderWoman.setAttribute('class', 'hero'); or wonderWoman.class = "hero";

Use classList to alter elements with mult classes

wonderWoman.classList.contains('hero');

wonderWoman.classList.add('warrior');

wonderWoman.classList.remove('warrior');

.toggle - adds class if element doesn’t have it already, remove class if it does have it, returns true if added and false if removed

Function to create an element - function createElement (tag,text) {const el = document.createElement(tag); el.textContent = text; return el} const aquaman = createElement('li','Aquaman');

heroes.insertBefore(aquaman,wonderWoman); inserts aquaman before wonderwoman

heroes.appendChild(wonderWoman); inserts at end of node list

.insertBefore() and .appendChild() can be used to move existing elements in list

heroes.removeChild(aquaman); removes element

Replace existing node - const h1 = document.getElementById('title'); const oldText = h1.firstChild; const newText = document.createTextNode('Justice League of America'); h1.replaceChild(newText,oldText);

The node lists returned by the document.getElementsByClassName() and document.getElementsByTagName() methods are live collections that will update to reflect any changes on the page.

Be careful when referring to elements by their index in a collection, as index can change

Use .style to change CSS - superman.style.border = "red 2px solid"; adds border

CSS property names that with dashes must be written in camelCase - superman.style.backgroundColor = 'blue';

.getComputedStyle() - returns style information for element

Best practice is to change style by adding classes and adding CSS in stylesheet

Javascript: Novice to Ninja Ch. 7 - Events

Event listeners perform code when a certain action on the page is taken.

The first parameter is the type of event, the second is the code to be performed - document.body.addEventListener("click", doSomething);

Can be called without a node, would apply to the whole page - addEventListener('click', () => alert('You Clicked!'));

.target - returns a reference to the node that fired the event

Find the position of where a mouse event occurs:

screenX and screenY properties show the number of pixels from the left and top of the screen respectively where the event took place

clientX and clientY properties show the number of pixels from the left and top of the client that is being used (usually the browser window)

pageX and pageY properties show the number of pixels from the left and top, respectively, where the event took place in thedocument

List of events - https://developer.mozilla.org/en-US/docs/Web/Events

Pressing the modifier keys such as Shift, Ctrl, Alt and meta(Cmdon Mac) will fire the keydown and keyup events, but not the keypress event as they don't produce any characters on the screen.

Avoid using preventDefault() to change default behavior. Users will expect certain behaviors, and preventing them may cause confusion.

When you click on an element, you are clicking on each element that it’s nested inside of

Bubbling (default) - the event fires on the element clicked on first, then bubbles up the document tree, firing an event on each parent element until it reaches the root node

Capturing - fires an event on the root element, then propagates downwards, firing an event on each child element until it reaches the target element that was clicked on
