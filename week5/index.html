Week 5 Notes

Javascript: Novice to Ninja Ch. 10 - Testing and Debugging

Exception - an error that when raised, produces a return value that can be used by the program to handle the error

Calling a function that does not exist raises a reference error

Stack trace - functions or methods that lead to the point of the error

Warning - error that doesn't cause code to crash

Strict mode - throws errors for things that can be seen as "poor style" - "use strict;"

Strict mode can also be applied inside of a function for use in only that function. That way, if you are working with other developers that aren't using strict mode, it won't throw errors for their code.

Linting enforces a style guide - JS Lint, JS Hint, and ES Lint are used to highlight sloppy programming practices and syntax errors

Determine browser support for a feature using feature detection by using an if statement to check whether an object or method exists before trying to call the method

Calling an method for an object that does not exist would throw an error, an if statement would return false and not execute

console.trace() method will log an interactive stack trace in the console

Other console methods - http://www.sitepoint.com/three-little-known-development-console-api-methods/

debugger - acts as a breakpoint when written into code

Error object can be created using constructor function with error message as parameter - const error = new Error('Oops, something went wrong');

Error objects used for specific errors:

RangeError - number is outside an allowable range

ReferenceError - reference is made to an item that doesn’t exist, like calling a function that hasn't been defined

SyntaxError - error in the code’s syntax

TypeError - error in the type of value used, like if a string is used when a number is expected

Error objects can also be used to create custom error objects - const error = new TypeError('You need to use numbers in this function');

You then "throw" the error object when an error occurs - throw new Error('Something has gone badly wrong!');

Try block - runs the code inside the block, if an exception occurs it will pass the error object that is thrown onto a catch block. The code inside the catch block will only run if an exception is thrown inside the try block

The error object is passed as a parameter to the catch block

finally block can be added after a catch block and will be executed after the try or catch block, whether an exception occurred or not
