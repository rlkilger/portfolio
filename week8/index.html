HTML5 & CSS3 for the Real World - CSS3 Transforms and Transitions

Transform - lets you lets you translate, rotate, scale, and/or skew elements

Transforms are applied in the order provided.

Translation - functions that allow you to move elements left, right, up, or down

Behaves similarly to position: relative; declare top and left, move elements up and down or left and right along the x and y axes - transform: translate(45px, -45px);

Can use translateX or translateY

Transforms donâ€™t work on inline elements

scale(x,y) - scales an element horizontally then vertically(if one value is provided, it is used for both x and y); grows and shrinks from center, so center stays the same.

transform: scale(2) - doubles font size; transform: scale(0.5) - cuts size in half

Can use scaleX or scaleY

When using transform to change element in different ways(scale and translate), put them in the same transform property - transform: translateX(40px) scale(1.5);

rotate() - rotates an element around the point of origin(default is center) by a specified angle value(positive moves clockwise, negative moves counter-clockwise) - transform: rotate(10deg);

skew(x,y) - skews text along the x and y axes(if one value given, it occurs on the x axis)

transform-origin - changes the point of origin(default is center - 50% 50%)

Transitions allow the values of CSS properties to change over time, essentially providing simple animations

Transitions work well when used with Javascript

Transition is declared in the default or originating state, the effect you want it to take is in the changing state(for example, hover)

transition-property - defines the CSS properties to be transitioned, with all for all properties being the default

If you want to transition more than one property, but not all, separate with comma

List of properties that can be animated - https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_animated_properties

transition-duration - how long the transition will take to complete the effect

transition-timing-function - ease, linear, ease-in, ease-out, or ease-in-out

ease - default; has a slow start, then it speeds up, and ends slowly

ease-in-out - similar to ease, but accelerates more sharply at the beginning

linear - animates at a constant speed

ease-in - slow to start but gains speed, then stops abruptly

ease-out - starts at full speed, then slows progressively

transition-delay - delays when the transition begins, useful for hovers to ensure that the user is intentionally hovering over the element

Negative delays start partway through the transition process

Transition shorthand - in the originating state, list them in order of property, duration, timing function, then delay, separated by a space - transition: transform 0.2s ease-out 50ms;

When applying it to 2 properties, if listing each effect separately, separate them by comma - transition-property: transform, color; transition-duration: 0.2s, 0.1s; transition-timing-function: ease-out, linear; transition-delay: 50ms;

When listing them using shorthand, list effect together, separate properties by comma - transition: transform 0.2s ease-out 50ms, color 0.1s linear 50ms;
